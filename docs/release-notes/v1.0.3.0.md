# v1.0.3.0 - Exact iteration rollback / 精确迭代回退

## English

Global iteration now creates an exact external snapshot before replacement. It covers every replaceable tracked and untracked file plus directory inventory and SHA-256, while excluding Git state, lifecycle/private state, credentials, and runtimes.

If replacement or later validation fails, an external restore runner reconstructs the pre-iteration worktree, removes iteration-added non-protected files, verifies hashes, refreshes global interfaces, records the failure, and stops. The owning error must be repaired before the complete iteration is rerun from the beginning. A rollback that cannot verify is a critical blocker.

## 中文对照

全局迭代现在会在替换前创建精确外部快照，覆盖所有可能被替换的受跟踪和未跟踪文件，并保存目录清单及 SHA-256；Git 状态、生命周期/私有状态、凭据和运行时始终排除。

若替换或后续验证失败，外部恢复执行器会重建迭代前工作区、移除迭代新增的非受保护文件、校验哈希、刷新全局接口、记录失败并停止。必须先修复责任模块，再从头运行完整迭代。无法验证的回退属于严重阻断。
