# v1.0.4.0 - Continuous iteration diagnosis / 持续迭代诊断

## English

File-organization and complete-global-iteration failures can now enter a named continuous-diagnosis mode. The controller attributes and records each failed probe, invokes an explicit safe owner repair, and restarts the entire selected probe until it succeeds.

The default has no arbitrary repair-attempt limit. Safety boundaries remain strict: an exact-rollback failure, a failed repair action, or work requiring new authority stops with a blocking error. Regression coverage verifies both file-organization and global-iteration owner routing and the fail-repair-retest sequence.

Error feedback also accepts local UTF-8 user-report files, preserving multilingual wording when a shell argument boundary cannot be trusted.

## 中文对照

文件整理与完整全局迭代的失败现在可以进入明确的“持续迭代诊断”模式。控制器会为每次失败探针记录责任模块和证据，调用显式的安全修复动作，并从头重跑所选模块的完整探针，直到成功。

默认不设置人为修复次数上限，但安全边界保持严格：精确回退失败、修复动作失败或需要新增授权的操作都会转为阻断错误。回归测试同时验证文件整理与全局迭代的责任路由，以及“失败—修复—重新测试”的完整序列。

错误反馈入口同时支持本地 UTF-8 用户报告文件，确保 shell 参数编码边界不可靠时仍能保留多语言原话。
