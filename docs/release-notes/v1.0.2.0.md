# v1.0.2.0 - Verified cleanup lifecycle / 经验证的清理生命周期

## English

Global self-iteration can now remove proven unnecessary content without treating filenames as sufficient authority. Only currently untracked allowlisted temporary/cache artifacts and truly empty non-protected directories are eligible. File candidates are copied to an off-root quarantine and SHA-256 verified before deletion. Candidate discovery is repeated on the active tree after replacement and again after validation, so regenerated caches are handled without trusting stale sandbox hashes.

Replacement is no longer the final step. The active repository must pass validation twice, its real global skill interfaces must be refreshed and validated, and aggregate cleanup/replacement evidence plus lifecycle state must be written back before the Git gate accepts the iteration.

## 中文对照

全局自身迭代现在可以删除经证明不必要的内容，但文件名本身不足以构成删除授权。只有当前未跟踪且在明确临时/缓存白名单中的文件，以及真正为空的非受保护目录才可进入清理。文件候选项会先复制到根目录外隔离区并通过 SHA-256 校验；替换后及验证后都会在当前目录重新发现候选，因此无需信任沙箱中的陈旧哈希即可处理再生缓存。

替换不再是最后一步。当前仓库必须连续通过两次验证，真实全局 Skill 接口必须刷新并通过验证，并且清理/替换聚合证据及生命周期状态完成写回后，Git 门禁才接受本次迭代。
