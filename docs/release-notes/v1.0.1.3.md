# v1.0.1.3 - Transactional isolated iteration / 事务式隔离迭代

## English

This maintenance release makes global self-iteration safe for the complete repository. It creates an isolated copy and an external recoverable backup, organizes every eligible non-protected file, repairs verified references, restores the canonical Git layout, runs full architecture and global-interface validation, and replaces the active system only after success. The pre-Git review gate now verifies that proof without reorganizing the active repository.

It also repairs the failures found during end-to-end testing: large archive timeout, quadratic reference scanning, Windows path normalization, line-ending drift, stale-reference false positives, null validation commands, moved restore-tool discovery, restoration of eligible untracked files, and portable selection of a PyYAML-capable validation runtime without a machine-specific path.

## 中文对照

本维护版本使全仓库范围的全局自身迭代具备事务安全性：先创建隔离副本和根目录外可恢复备份，再整理全部合格的非受保护文件、修复已验证引用、恢复 Git 规范布局、运行完整架构及全局接口验证，并且仅在成功后替换当前系统。Git 前复核门禁现在只验证该证明，不再整理当前仓库。

本版本同时修复端到端测试发现的问题：大归档超时、二次复杂度引用扫描、Windows 路径规范化、换行漂移、陈旧引用误报、空验证命令、恢复工具被移动后的发现问题、合格未跟踪文件的完整恢复，以及无需机器专属路径即可选择支持 PyYAML 的验证运行时。
