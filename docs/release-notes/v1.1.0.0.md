## What's new / 本次更新

### English

- Add Brainstorming Lite for turning consequential ambiguity into an evidence-grounded design before execution.
- Clarify complete experience-system iteration routing so authorized candidate processing is handled by the outer controller.
- Add regression checks for the new collaboration and planning contracts.
- Reset current private release metadata and associated user-facing files to `1.1.0.0 / v1.1`.
- Include the verified global-iteration speed optimizations in the release record: protected ignored roots are filtered before rollback/cleanup inventory, candidate reports avoid full JSON parsing when no `status: candidate` marker is present, and the complete replacement proof remains gated by rollback, validation, and interface checks.
- Add autonomous `caveman-lite` routing for low-risk routine chat progress and repetitive experience-system work, while keeping full prose for safety boundaries, formal reports, release notes, error feedback, and ordered user instructions.
- Add autonomous `brainstorming-lite` routing for material unresolved workflow, skill, learning, experience, or architecture choices, without making bounded reversible work wait for mandatory interviews.
