# Academic Research Workflow

## Inputs

- Research question and desired deliverable.
- Source locators: paper, PDF, DOI, arXiv, LaTeX, repository, commit, or dataset scope.
- Audience, privacy constraint, and available specialist skills.

## Steps

1. Select one mode: PaperRead, EvidenceSynthesis, ManuscriptDraft, ResearchCode, or FigurePlan.
2. Create a project-local manifest with source locators and expected deliverables.
3. Map observations to stable source locations and mark them supported, disputed, pending, or out-of-scope.
4. Route source extraction, citations, prose, code mutation, layout, and figures to their owners.
5. Deliver the requested artifact with unresolved limits kept separate from findings.

The manifest is the headless control artifact. Existing global modules implement
the selected route; a missing interactive UI does not mean the function is
unavailable.

## Verification

- Every material source-grounded claim has a locator or remains pending.
- No private source text, credential, or raw data enters the manifest or global knowledge.
- No specialist owner is bypassed for a structural write or external action.
