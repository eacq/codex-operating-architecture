# Local Experience Iteration Workflow

## Inputs

- Project lifecycle files, pending events, and structured error reports.
- Metadata-only conversation catalog and memory indexes.
- Global experience ledger and linked knowledge index.

## Steps

1. Read project authority and enumerate pending or candidate evidence.
2. Refresh derived metadata using its owning generator.
3. Validate UTF-8 JSON with the creating runtime before diagnosing data corruption.
4. Compare observations with lifecycle lessons, the ledger, and knowledge links.
5. Classify each item as verified lesson, project candidate, stale derived artifact, or no action.
6. Read a matched raw session only when metadata cannot resolve a specific question.
7. Promote only repaired, repeated, or independently corroborated rules.

## Verification

- Derived catalogs parse with their creating runtime.
- No raw session payload, credential, or machine-specific private path enters Git knowledge.
- Every promoted rule has scope, source, verification, and invalidation conditions.
