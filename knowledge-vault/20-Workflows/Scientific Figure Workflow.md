# Scientific Figure Workflow

Map the claim, target journal, data, and visual hierarchy before rendering. Reuse `academic-figure-generation` and its verified PaperBanana runtime when suitable. Use [[Academic Figure Optimizer Module]] when the deliverable is a quantitative or engineering figure that must preserve data fidelity, physical dimensions, dpi, fonts, and export formats. Validate both the Codex runtime and project environment. Provider routing must respect [[Provider Capability Boundary]]. Feed final figure meaning back into [[Academic Writing and Citation Workflow]].

For knowledge notes, prefer Mermaid when relationships are structural. Use `codex-image-workflow` for generated or licensed bitmaps, Bilibili hosting, remote-link replacement, and local cleanup.
