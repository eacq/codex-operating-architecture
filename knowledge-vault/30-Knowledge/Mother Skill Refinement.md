# Mother Skill Refinement

Mother skill refinement applies when a broad owner is still correct but its
entrypoint has become too procedural. The parent skill remains the public
discovery surface and keeps ownership, routing, safety boundaries, and
validation summary. Detailed gates and modes move into owner-internal subskills.

This differs from creating a new top-level module: no registry entry changes
unless independent evidence later proves a separate trigger, artifact set,
maintained knowledge base, and safety boundary. It also differs from generic
documentation extraction because every moved procedure remains directly linked
from the parent before use.

The first application is `codex-self-evolution`, whose global iteration and
publication gates now live in internal subskills while the mother controller
stays concise.
