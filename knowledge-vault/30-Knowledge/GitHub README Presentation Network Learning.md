## Adopted practice

Choose audit, whole-README, or asset-only scope before edits; ground the first
screen in real project proof; preview before Git publication; preserve a clear
Markdown reading path when images fail.

## Local adaptations and exclusions

Use a format-selection gate instead of assuming SVG. Keep third-party cards,
counters, tracking, profile workflows, GIFs, attribution, pushes, PRs, and
showcase submission opt-in. The local owner is `codex-task-execution` because
the workflow's execution, verification, and publication boundaries overlap
that owner; no new top-level module is needed.

## Validation and invalidation

Validate with a bounded README/profile audit or asset-only preview, plus the
global skill validator. Reassess if GitHub rendering restrictions, upstream
licenses, the installed workflow's safety contract, or local visual-format
policy materially changes.
