# Project Creation Capability

Project creation is not only making files. It is the ability to convert a
folder, user goal, local evidence, and prior history into a project-specific
operating system for Codex.

## Definition

A project is considered created when it has:

- lifecycle state;
- evidence-backed requirements;
- repeatable workflows;
- verified project experience;
- append-only retrospectives;
- storage/index rules for durable artifacts;
- runtime and environment preflight;
- error-feedback routing for repeated failures;
- promotion boundaries for global knowledge.

## Boundary

Project facts remain in the project. The global experience system receives only
verified, reusable, non-duplicate rules with scope and invalidation conditions.

## Example

XLW1 converted manuscript-formatting work into:

- project-local lifecycle files;
- domain QA scripts;
- project-local skills;
- version indexes;
- runtime/encoding preflight;
- error-feedback reports;
- verified retrospectives.

The reusable part is the project-creation pattern, not the manuscript-specific
formatting details.
