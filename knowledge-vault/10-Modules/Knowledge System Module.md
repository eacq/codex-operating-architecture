# Knowledge System Module

Owns Obsidian-compatible notes, typed links, generated indexes, graph validation, MindMaster-compatible map views, Codex-local learning, and user Anki export. [[Learning Audience Boundary]] prevents internal operating rules from leaking into the user's study deck. [[Experience and Knowledge Architecture]] summarizes the end-to-end responsibility boundary. The module consumes [[Verified Experience Promotion]] and preserves [[Project Knowledge Boundary]]. [[Mind Map Knowledge Workflow]] defines how maps remain derived from Markdown rather than becoming a second source of truth. When a bitmap view materially helps, it routes generation, editing feedback, hosting, and local cleanup through [[Image Workflow Module]] instead of storing unmanaged images. It supports [[Research Knowledge Module]] without duplicating its domain workflows.
