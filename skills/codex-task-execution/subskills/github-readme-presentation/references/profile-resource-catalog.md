# GitHub Profile Resource Catalog

Source: `rzashakeri/beautify-github-profile`, CC0-1.0, commit
`34f41124c4b32fa9b4ae9504966a1d0f46080aaf`.

Use this as a discovery catalog, not as a mandate to fill every profile with
dynamic widgets. Its useful component families are badges, icons, profile
generators, contribution displays, and concise profile README examples.

Selection rules:

- Add a component only when it helps a visitor understand the person's work,
  current focus, or a verifiable project path.
- Prefer static Markdown, a maintained local asset, or a stable official
  GitHub surface over a third-party counter, tracking endpoint, or opaque
  remotely generated card.
- Confirm that a linked service is live and suitable before embedding it;
  retain meaningful Markdown and alt text if it fails.
- Keep the profile readable without images. Avoid repetitive badges, unrelated
  social proof, sensitive activity data, and decorative animation.
- Profile repository creation, external account linking, workflow tokens, and
  publishing remain user-authorized GitHub actions.
