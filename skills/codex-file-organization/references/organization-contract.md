# Organization Contract

- Global architecture iterations automatically apply the safe managed-root plan only inside an isolated copy. The executor creates an off-root recoverable backup, organizes eligible files, updates exact verified references, restores the canonical Git layout, quarantines proven disposable artifacts, and validates the sandbox; the active system is replaced only after every check passes. Replacement is followed by two active-repository validations, real global-interface refresh/validation, and lifecycle writeback.
- Folder roles: `00-inbox` unclassified intake; `10-active` current work; `20-reference` durable inputs; `30-output` derived deliverables; `40-archive` closed material; `90-private-local` non-versioned local-only material.
- Do not move `.git`, `.codex`, `.env*`, `auth.json`, secrets, browser profiles, encrypted stores, or any path outside the selected root.
- Store any name-bearing `organization-plan.json` and backup manifests only in a selected local project location; do not commit them. Lifecycle records committed or shared outside the project may contain only aggregate bucket counts, policy version, checksums, and decision status.
- Keep source filenames unless a rename is explicitly approved. Resolve collisions by adding a suffix rather than overwriting.
- Evolve the taxonomy with the same evidence rule as workflows: retain/refine for a proven rule, add only for a repeated unserved use case, merge substantially overlapping buckets, split a bucket only when lifecycle or access controls materially differ, and deprecate before removal. Test handoffs to workflow, knowledge, experience, backup, and visual documentation before an economy pass.
- `file-organization.json` is the project-local policy. `file-organization-review.json` is a redacted lifecycle result. `file-organization-backup.json` records only backup readiness and a backup checksum; all are under `.codex/project/` by default.
- The global automatic managed root is `.`: all non-backup, non-protected project files are considered. `.git`, `.codex`, credential files, and external backup roots stay excluded. Before validation and Git, tracked and eligible untracked files plus affected configuration/reference files are restored from the local move manifest. Failure keeps the active system unchanged and returns the isolated copy to repair/retry.
- All non-protected folders are inventoried and impact-reviewed on each iteration. `managed_roots` selects which folders may be automatically reorganized; use a project-local credential reference only for an optional remote backup target. Store the secret itself in Windows Credential Manager or DPAPI, never in JSON, Git, logs, a backup manifest, or an image prompt.
- Automatic cleanup is allowlist-based, never inference-only: currently untracked `*.pyc`, `*.pyo`, `*.tmp`, `*.temp`, `*.bak`, `*.orig`, `*.rej`, editor backups, platform litter, files under known language-test cache directories, and empty non-protected directories are eligible. Recompute candidates after replacement and after validation instead of propagating a stale pre-validation hash. Every file is hash-verified in an off-root quarantine before its deletion. Tracked files and protected/runtime/dependency trees are ineligible regardless of name. For global iteration, discover disposable files through Git's untracked plus ignored path sets, then use candidate-parent empty-directory cleanup; this preserves safety while avoiding a broad scan over all tracked project files.
- First-party temporary and scratch locations are not free-form. Resolve them through `$ARCHITECTURE_ROOT\scripts\Resolve-CodexRunRoot.ps1`; default disposable work lives under `.runtime\tmp`, longer-lived candidate work under `.runtime\work`, and reusable caches under `.runtime\cache`. These roots are ignored local state and may be cleaned with a current manifest. OS temp and user-home caches are exception paths, not defaults.
- Before replacement, create an exact off-root rollback snapshot of all replaceable tracked and untracked files plus directory inventory and SHA-256. Exclude `.git`, `.codex`, `.runtime`, validation homes, dependency/runtime/private-local roots, credentials, and secret-shaped files from both replacement and rollback. The optimized implementation builds the manifest from Git tracked, untracked, and ignored path authority, copies the privacy-filtered tree with robocopy, and verifies each manifest file hash in the copied snapshot. On post-replacement failure, use the external snapshot runner rather than the possibly broken active script, restore and verify the snapshot, remove iteration-added non-protected files, refresh global interfaces, write a local failure record, then stop for owner repair and a full rerun. If rollback cannot verify, keep lifecycle pending and report a critical error.
- Continuous diagnosis is the explicit testing/debugging mode for file organization and complete global iteration. It runs the selected owner probe, records the failed attempt, invokes an explicit safe repair action, and restarts the entire probe until it passes. Unlimited means no arbitrary attempt count, not permission to ignore a failed rollback, install software, alter credentials, publish externally, or continue after a repair action fails.
