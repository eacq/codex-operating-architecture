 # Normalized requirement contract

Use this contract for material, multi-owner, or global-system work. Keep the
user's literal goal distinct from inferred implementation choices.

Required fields:

- Goal and intended user-visible outcome.
- Constraints and authority boundaries.
- Assumptions that may be revised without changing the goal.
- Exclusions that prevent scope creep.
- Project-local vocabulary only when a term has repeated ambiguous use.
- Consequential decisions only when alternatives materially affect delivery, safety, or verification.
- Observable acceptance criteria.
- Validation evidence and stop condition.

The brief is execution-ready only when each acceptance criterion has a matching
validation method. A brief never authorizes external installation, credential
access, destructive mutation, paid action, or publication.
