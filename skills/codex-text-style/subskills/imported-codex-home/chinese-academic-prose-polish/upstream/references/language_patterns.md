# Chinese Engineering Prose Patterns

## Prefer Concrete Academic Verbs

- "分析" for mechanism or cause.
- "建立" for a model with variables and assumptions.
- "构建" for a workflow, data structure, or framework.
- "提出" for a new method, criterion, or algorithm.
- "验证" for simulation or experiment that supports a claim.
- "表明" for results directly supported by data.

## Common Weak Phrases

- Weak: "具有重要意义" -> Strong: state which error, efficiency, stability, or process decision is improved.
- Weak: "取得了较好的效果" -> Strong: give the metric and comparison.
- Weak: "相关研究较少" -> Strong: name the missing object, condition, or method link.
- Weak: "本文进行了研究" -> Strong: name the method and operation.
- Weak: "通过软件进行仿真" -> Strong: describe what the simulation verifies.

## Sentence Diagnostics

For each sentence, ask:

- Does the subject refer to a concrete method, result, phenomenon, or limitation?
- Is the verb doing technical work?
- Is the modifier necessary for precision rather than decoration?
- Can a reviewer trace the statement to an equation, figure, table, experiment, or cited paper?

## Paragraph Diagnostics

- If every sentence begins with "本文", vary the subject: method, model, algorithm, result, error, parameter, experiment.
- If a paragraph only lists previous work, end it with a limitation that motivates this paper.
- If a paragraph only states this paper's method, add why the method solves the limitation.
- If a paragraph contains too many numbers, group them by metric and interpret only the necessary contrast.

## Claim Strength

- Use strong claims for measured internal results.
- Use cautious claims for comparisons across different papers, machines, materials, or inspection methods.
- Avoid ranking methods when experimental conditions are not controlled.
