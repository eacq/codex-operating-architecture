# thesis-polish

一个专注毕业论文生产的 Claude Code skill，针对中文理工科毕业论文的写作风格调整和内容审校。目标是让论文读起来是有脑子的活人研究生写的。

专注毕业论文这场景。60 多条规则、两个核心自检问题和一套基于范例论文对照的工作流。你值得信赖。

## 为什么做这个

用 Claude 写中文毕业论文的时候，一会是 AI 味太重，满篇"值得注意的是"、"综上所述"、"其一其二"，动不动冒号拖尾和破折号，答辩老师一眼就看出来。一会又是为了躲 AI 味走到另一个极端，写得太口语、太浅，低于毕业论文应有的水准。

毕业论文是一个非常特定的文体。它不是期刊论文，不是综述），更不是工作报告。它讲究的是把工作**按特定的结构和语感**呈现给答辩评委。。

这个 skill 就瞄准这一件事：让 Claude 写出来的、或帮你改出来的，是**真正像毕业论文的毕业论文**。

本skill是融合了本人在写整个毕业论文过程中所有1G的与CLAUDE斗智斗勇所凝结出来的规则和经验。

## 最关键的一点：以一本范例论文为锚

这是整个 skill 最该用好的地方。

规则能抓大部分 AI 痕迹，但"像不像毕业论文"最终还是要靠具体的对照。不同学科、不同学校、不同导师偏好的毕业论文风格差异很大，所以 skill 让你自己指定一本**同方向、你信任的范例论文**（最好是你读过、觉得语言扎实、目录清楚、答辩通过的那一类）。

配上范例后，skill 会从范例 PDF 里抽出一个五到十 KB 的"风格指纹"，装章引言样例、本章小结样例、图题、高频句式、小节命名风格这些。之后每次对话只读这个小指纹，不碰原文，省 token。

写或改的时候 skill 会反复自问两件事：

- 这句话删掉或改短，意思有损失吗？没有就是 AI 填充。
- 这句话、这个小节名，有可能出现在你的范例论文里吗？不可能就是不合格。

第二条就是范例论文要做的事。没有它，规则只能判"这段像不像 AI 写的"；有它，才能判"这段像不像**你这个方向的毕业论文**"。

配置很简单，对话说明：

```
参考论文是 /path/to/some_thesis.pdf
```

skill 会自动验证路径、存下来、抽指纹。换论文、删论文、加论文都是同样一句话的事。

不配置也能跑，会变成纯规则模式。

## 装好

```bash
git clone https://github.com/<your-username>/thesis-polish.git ~/.claude/skills/thesis-polish
```

或者把整个目录放到 `~/.claude/skills/thesis-polish/` 下就行。

## 怎么用

```
/thesis-polish 审一下 chapters/chapter3.tex
```

也可以不打斜杠，直接说：

```
用 thesis-polish 审 chapters/chapter3.tex 的 3.2 节，给诊断报告
```

```
按 thesis-polish 的规则帮我写 4.1 节的引言
```

说"帮我写"会走写作模式，Claude 按规则下笔，不是写完再改。说"审"、"润色"、"看看哪里 AI 味重"会走审校模式，出诊断报告或直接改写。说"快速过一遍"、"只看硬伤"就只扫 20 条硬规则。



## 规则都在哪

| 文件 | 管什么 |
|------|--------|
| `SKILL.md` | 入口、工作流、20 条硬规则 |
| `meta-rules.md` | 讨论内容别写进正文、别暴露 AI 痕迹、别为改而改 |
| `patterns-vocabulary.md` | 不要用的词，比如"值得注意的是"、"综上"那一堆 |
| `patterns-syntax.md` | 不要用的句式，冒号拖尾、其一其二之类 |
| `patterns-layout.md` | 破折号、加粗、emoji、图表和公式排版 |
| `patterns-structure.md` | 小节怎么命名、章引言和本章小结怎么写 |
| `patterns-math.md` | 公式前后文字怎么接，假设、定理、Remark 怎么写 |
| `reference-thesis.md` | 你的范例论文配置，空模板等你填 |

## 20 条硬规则

定稿前可以拿来过一遍。

1. 不用破折号 ——
2. 不用"一是/二是"、"其一/其二"列举
3. 不用"X：Y"式的冒号拖尾句
4. 不用"值得注意的是"、"综上所述"、"首先……其次……最后"这种套话
5. 不用"进行+名词"堆砌，"进行分析"改"分析"
6. 不用"原因在于"
7. 不要先报框架再说内容，"本节从 X、Y、Z 三方面展开"这种别写
8. 定理前不写铺垫段
9. 不加粗、不给公式画外框、不用 emoji
10. 图题要短，子图 (a)(b)(c) 的说明放正文不放图题
11. 小节标题不机械拼接、不用自造复合术语
12. 缩写第一次出现要给全称
13. 摘要不堆缩写、不用中文冒号、不用破折号串路线
14. 公式和正文之间留空行
15. 证明不"分三部分"，不(1)(2)(3)
16. 长句控制在 60 字以内
17. 数字就是结论，别前置"可以看出"，别后置"验证了有效性"
18. 和 Claude 讨论过的话不要直接抄进正文
19. 改写要克制，原句通顺就不动
20. 本章小结一段话写完，不分条

## 一个改写示例

改之前：

> 综上，本方法在多个方面表现出显著优势：其一，计算复杂度较低；其二，对参数摄动具有较强的鲁棒性；其三，易于工程实现。值得注意的是，该方法的核心机制在于——通过将约束编码为障壁函数——使得优化问题在保证可行性的同时实现了性能与鲁棒性之间的有效平衡。

命中了"综上"、"显著优势"、"其一其二其三"、破折号、"值得注意的是"、"核心机制"、"有效平衡"，几乎把 AI 味词汇和句式凑齐了一轮。

改之后：

> 本方法的计算复杂度较低，对参数摄动也较鲁棒，工程实现简单。优化问题通过将约束编码为障壁函数保持可行性。

## 有哪些做不到

`.tex` 文件处理得比较保守，LaTeX 指令和公式内部不会动，只改中文正文。首次抽指纹要读一次 PDF 原文，大概 30 到 50K token，这个省不了，但之后每次都吃指纹。英文摘要不管，那部分建议配 humanizer 一类的工具用。诊断报告给建议但默认不直接改，因为改多了会把作者自己的语感改没，"为改而改"比 AI 味还糟。

## 这个 skill 是怎么来的

一个写毕业论文的朋友用 Claude 起草和改稿的时候，反复和模型交涉"这样不像毕业论文"、"这里 AI 味太重"、"小节名改改"。这些散落的吐槽和修正攒了几百条。我把这些按主题归类、去重、提炼，变成了现在的 60 多条规则、两个核心自检问题和一套基于范例论文对照的工作流。

跑下来的体会是：中文毕业论文里"像不像活人写的"其实不玄，基本就是一组可以列举的模式黑名单，加上一本具体的范例论文做风格锚。
